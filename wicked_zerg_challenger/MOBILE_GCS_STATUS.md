## ? 권장사항

### 옵션 1: PWA (Progressive Web App) 구현 (권장) ?

**장점:**
- Python 프로젝트 내에서 구현 가능
- Android/iOS 모두 지원
- 네이티브 앱처럼 사용 가능 (홈 화면에 추가, 오프라인 지원 등)
- 별도 앱 스토어 등록 불필요

**구현 방법:**
1. `manifest.json` 파일 추가 (앱 메타데이터)
2. Service Worker 추가 (오프라인 지원, 푸시 알림 등)
3. 모바일 최적화 CSS/JavaScript 개선
4. HTTPS 지원 (PWA 필수 요구사항)

**결과:**
- 모바일 브라우저에서 "홈 화면에 추가" 가능
- 네이티브 앱처럼 실행 가능
- 오프라인에서도 기본 기능 사용 가능

---

### 옵션 2: ngrok 터널링 통합 (선택적)

**장점:**
- 외부 네트워크(LTE/5G)에서 로컬 서버 접근 가능
- 별도 포트 포워딩 설정 불필요
- 보안 연결 지원 (HTTPS)

**구현 방법:**
1. ngrok Python SDK 또는 subprocess 통합
2. 자동 터널링 시작/종료 스크립트
3. 터널 URL 자동 업데이트 (대시보드에 표시)

**주의사항:**
- ngrok 계정 및 API 키 필요
- 무료 계정은 세션 시간 제한 있음
- 유료 플랜은 추가 비용 발생

---

### 옵션 3: Android 네이티브 앱 개발 (복잡)

**요구사항:**
- 별도 Android 프로젝트 (Java/Kotlin)
- Android Studio 개발 환경
- API 통신 (REST API 또는 WebSocket)
- 앱 스토어 배포 (선택적)

**권장하지 않음:**
- 현재 프로젝트 범위를 벗어남
- Python 프로젝트와 별도로 관리 필요
- 개발 및 유지보수 비용 증가

# Swarm-Net Airspace Manager
### 군집 드론 기반 사용자 공역 통제 및 알림 시스템

---

## 1. 프로젝트 개요 및 배경

### 1.1 배경

도심 항공 모빌리티(UAM) 시대가 도래하면서, 저고도 공역에서의 드론 운용이 폭발적으로 증가하고 있다. 택배 배송, 농업 방제, 인프라 점검, 측량 등 다양한 목적의 드론이 동일한 공역을 공유하게 되면서, **실시간 공역 통제(Airspace Control)**의 필요성이 대두되었다.

기존의 유인 항공기 관제(ATC) 시스템은 고정 레이더 기지국에 의존하지만, 이는 저고도 소형 드론 탐지에 한계가 있다. 본 프로젝트는 **"레이더 자체를 드론으로 대체"**하는 발상의 전환을 제안한다.

### 1.2 핵심 아이디어

> **군집 드론(Swarm Drones)이 스스로 이동식 레이더망을 형성하여, 특정 공역을 실시간으로 감시하고 관리한다.**

- **Swarm Fleet**: 6~12대의 군집 드론이 다각형 대형을 이루며 3D 공간에 가상의 '레이더 돔(Dome)'을 전개
- **Mesh Radar**: 드론 간 LiDAR + RF 통신으로 Mesh Network를 구축, 내부 진입 객체를 실시간 탐지
- **Timer-Based Control**: 탐지된 사용자 드론에 체공 시간을 할당하고, 만료 시 자동 복귀 명령

### 1.3 기술 원천: SC2 Swarm Control

본 시스템의 군집 제어 알고리즘은 StarCraft II 게임봇에서 개발·검증된 **Boids Swarm Control** 알고리즘을 실제 드론 환경으로 전이(Transfer)한 것이다.

| SC2 컴포넌트 | Drone ATC 매핑 | 전이 신뢰도 |
|---|---|---|
| Blackboard (중앙 상태관리) | Flight Data Hub | Direct |
| Boids Algorithm (군집 이동) | Formation Flight Control | Direct |
| Authority Mode (우선순위) | ATC Priority Levels | Direct |
| IntelManager (탐지) | Sensor Fusion (LiDAR+GPS) | Adapt |
| CreepManager (영역 확장) | Airspace Corridor | Concept |

## 2. 핵심 시스템 로직

### 2.1 Phase 1 — 통제 구역 설정 및 레이더망 형성

```
관제 서버 → [GPS Waypoints 할당] → 군집 드론 출격
→ 다각형 대형(Polygon Formation) 전개
→ Mesh Network 활성화
→ 레이더 돔(Dome) ACTIVE
```

- 관제 서버가 통제 대상 공역의 GPS 좌표를 군집 드론에 할당
- 드론들은 Boids 알고리즘(Separation + Alignment + Cohesion)에 따라 다각형 꼭짓점 위치로 이동
- 상호 간 LiDAR 및 RF 통신 링크를 수립하여 Mesh Radar Network 전개
- 모든 노드 연결이 확인되면 통제 공역이 **ACTIVE** 상태로 전환

### 2.2 Phase 2 — 사용자 드론 탐지 및 식별

```
사용자 드론 진입 → [RF 스캔 + MAC 추출]
→ 관제 서버 DB 등록
→ 비행 허가 검증
→ 인증 ACK 응답
```

- 레이더망 경계선을 통과하는 객체를 RF 신호 스캔으로 즉각 탐지
- 고유 식별자(MAC 주소, Transponder ID)를 추출하여 관제 DB에 실시간 등록
- 사전 등록된 비행 계획(Flight Plan)과 대조하여 허가/거부 판정
- 미등록 드론은 즉시 **UNKNOWN** 태그로 분류하여 우선 감시 대상으로 지정

### 2.3 Phase 3 — 체공 시간 할당 및 실시간 추적

```
타이머 시작 (예: 15:00) → 매 1초 위치 동기화
→ 삼각측량 (X, Y, Z)
→ 대시보드 실시간 표시
```

- 사용자 드론 인증 완료 시 사전 설정된 비행 허가 시간 카운트다운 시작
- 군집 드론 3대 이상이 삼각측량(Triangulation)으로 실시간 3D 위치 연산
- 1초 주기로 관제 대시보드에 위치 + 잔여 시간 데이터 Push
- 상태 분류:
  - **🟢 NORMAL**: 잔여 시간 > 3분
  - **🟡 WARNING**: 잔여 시간 ≤ 2분
  - **🔴 EXPIRED**: 잔여 시간 = 0

### 2.4 Phase 4 — 경고 알림 및 퇴각 통제

```
잔여 2분 → [1차 경고 Push]
→ 잔여 0초 → [최종 경고 + 복귀 명령]
→ 미응답 30초 → [긴급 프로토콜 발동]
```

| 시점 | 조치 | 알림 유형 |
|---|---|---|
| 잔여 2분 | 비행 시간 임박 경고 | 🟡 Push 알림 |
| 잔여 0초 | 비행 종료 + 즉시 복귀 명령 | 🔴 강제 알림 |
| 미응답 30초 | 긴급 프로토콜 (물리적 에스코트/차단) | 🚨 관제 개입 |

## 4. 기대 효과 및 활용 분야

### 4.1 기대 효과

| 항목 | 기존 방식 | Swarm-Net 방식 |
|---|---|---|
| **탐지 범위** | 고정 레이더 반경 내 | 군집 이동으로 동적 공역 커버 |
| **배치 시간** | 인프라 설치 수 개월 | 드론 출격 수 분 내 전개 |
| **비용** | 고정 레이더 수억 원 | 드론 fleet 운영비 |
| **유연성** | 고정 위치 | 실시간 공역 재구성 가능 |
| **확장성** | 추가 기지국 필요 | 드론 추가 투입으로 즉시 확장 |

### 4.2 활용 분야

1. **불법 드론 통제**: 공항, 군사 시설 주변의 비인가 드론 감시 및 퇴각 유도
2. **에어쇼/이벤트 안전 관리**: 대규모 드론 쇼 시 관람 드론의 안전 거리 유지
3. **재난 현장 공역 통제**: 소방/경찰 드론 운용 시 민간 드론 진입 차단
4. **도심 UAM 공역 관리**: 드론 택시, 화물 드론의 복층 비행로(Corridor) 관리
5. **농업 방제 구역 관리**: 방제 작업 중 해당 공역의 일반 드론 진입 제한
6. **군사 작전 구역 설정**: 전술 지역의 동적 비행 금지 구역(No-Fly Zone) 운용

### 4.3 기술 발전 로드맵

```
Stage 1: SC2 시뮬레이션     → 알고리즘 검증 (10,000+ 게임)
Stage 2: 3D 시뮬레이터      → 파라미터 적응 (2D→3D)
Stage 3: 실드론 테스트       → 5대 편대 비행 검증
Stage 4: 도시 스케일 ATC    → 100+ 드론 관제 시스템
```

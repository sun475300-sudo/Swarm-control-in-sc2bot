## ? 파라미터 의미 및 분석

### 1. **lair_supply: 12**
- **의미**: 라이어(2단계 본진)를 건설하는 시점의 인구수
- **분석**: 매우 빠른 라이어 타이밍 (일반적으로 14-16)
- **전략**: 빠른 테크 업그레이드를 통한 고급 유닛 조기 생산

### 2. **gas_supply: 17**
- **의미**: 가스 채취를 시작하는 시점의 인구수
- **분석**: 표준적인 타이밍 (일반적으로 16-18)
- **전략**: 적절한 타이밍에 가스 확보

### 3. **spawning_pool_supply: 17**
- **의미**: 산란못을 건설하는 시점의 인구수
- **분석**: 표준적인 타이밍 (일반적으로 14-18)
- **전략**: 기본 유닛 생산 시설 확보

### 4. **natural_expansion_supply: 30**
- **의미**: 자연 확장 기지를 건설하는 시점의 인구수
- **분석**: 표준적인 확장 타이밍 (일반적으로 28-32)
- **전략**: 경제 확장을 통한 자원 수급 증가

### 5. **roach_warren_supply: 56**
- **의미**: 바퀴 소굴을 건설하는 시점의 인구수
- **분석**: 중반 전투 유닛 생산 시설
- **전략**: 중반 전투력 확보

### 6. **hive_supply: 12**
- **의미**: 하이브(3단계 본진)를 건설하는 시점의 인구수
- **분석**: ?? **비정상적으로 빠름** (일반적으로 80-100+)
- **주의**: 이 값은 학습 오류일 가능성이 높음
- **권장**: 재학습 또는 수동 조정 필요

### 7. **hydralisk_den_supply: 122**
- **의미**: 히드라리스크 굴을 건설하는 시점의 인구수
- **분석**: 후반 대공 유닛 생산 시설
- **전략**: 후반 대공 전투력 확보

## ? 학습 데이터 출처

### 학습 방법
1. **리플레이 분석**: 프로 게이머 리플레이 파일에서 빌드 오더 추출
2. **통계 분석**: 여러 리플레이의 평균값 계산
3. **가중치 적용**: 승리 리플레이에 더 높은 가중치 부여

### 학습 소스
- **리플레이 디렉토리**: `D:/replays/replays` (Zerg 프로 게이머 리플레이)
- **학습 횟수**: 최대 100개 리플레이 분석
- **저장 위치**: `local_training/scripts/learned_build_orders.json`

## ?? 주의사항 및 개선 필요 사항

### 1. **hive_supply: 12** - 비정상 값
- **문제**: 하이브는 보통 80-100+ 인구에서 건설됨
- **원인**: 학습 데이터 부족 또는 리플레이 분석 오류
- **해결**: 
  - 더 많은 리플레이 학습 필요
  - 수동으로 기본값(80-100)으로 조정 권장

### 2. **모델 파일 부재**
- **현재 상태**: `local_training/models/` 디렉토리가 존재하지 않음
- **의미**: 신경망 모델이 아직 학습되지 않았거나 저장되지 않음
- **다음 단계**: 
  - 게임 실행 후 모델이 자동 생성됨
  - 학습 파이프라인 실행 시 모델 저장됨

## ? 학습 데이터 사용 현황

### 게임 실행 시 자동 적용
1. **빌드 오더 파라미터**: `config.py`의 `get_learned_parameter()` 함수를 통해 자동 로드
2. **신경망 모델**: `zerg_net.py`의 `ReinforcementLearner`가 자동으로 모델 로드 시도

### 적용 위치
- **production_manager.py**: 빌드 오더 타이밍 결정 시 사용
- **economy_manager.py**: 확장 타이밍 결정 시 사용
- **wicked_zerg_bot_pro.py**: 전략 결정 시 신경망 모델 사용

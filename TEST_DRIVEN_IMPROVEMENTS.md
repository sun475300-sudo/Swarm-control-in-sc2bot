# Test-Driven Improvement List

Generated by running test suites and inspection on the SC2 commander bot
(`Swarm-control-in-sc2bot` / `wicked_zerg_challenger`).

## Findings

### Test infrastructure
- `comprehensive_test_suite.py` returns **hard-coded** pass/fail numbers
  instead of actually running its sub-suites. The "47.3%" pass-rate it
  reports has no relation to current code.
- `fuzz_test.py`:
  - `fuzz_unit_positions`: random range is `-1000..1000`, but anything with
    a negative coordinate is counted as a *failure*, not as the validator
    correctly rejecting bad input. ~50% of calls always look "failed".
  - `fuzz_resource_values`: the success condition compares pre-sanitized
    inputs to post-sanitized values, so the test fails any time the
    sanitizer does its job.
  - `fuzz_unit_names`: ~50% of names are random garbage, all reported as
    failures even though the validator correctly rejects them.
  - `fuzz_build_orders`: never resets `valid` once an invalid order was
    seen; results are flaky.
- Every "fail" is actually the validator working as intended; the test is
  inverting success/failure.

### Module / import-time bugs
- `wicked_zerg_challenger/scouting/advanced_scout_system_v2.py` defines an
  empty fallback `class UnitTypeId: pass` but uses
  `UnitTypeId.OVERLORD` as a default arg, so importing the module raises
  `AttributeError` when the real `sc2` package is unavailable.
- `wicked_zerg_challenger/check_learning_rate.py` runs script-level code
  (incl. `sys.stdout = io.TextIOWrapper(...)`) **on import** because there
  is no `if __name__ == "__main__":` guard. Importing the module replaces
  stdout for whoever imports it.
- `wicked_zerg_challenger/check_proxy.py` calls `sys.exit()` at import
  time, crashing any aggregator that imports the package.
- 35 modules redefine fallback `UnitTypeId` classes individually. Drift is
  inevitable; we should consolidate to a single shared stub.

### Bot logic / SC2Coach
- `sc2_coach.py` only ships 10 patterns and is missing common Zerg
  problems (queen production, mineral patches drained, no anti-air,
  missing creep spread, etc.).
- No unit-tests exist for `SC2Coach.get_coaching_advice` even though the
  bot relies on it.

## Improvement Plan (batches)

### Batch 1 — Test infrastructure
- [ ] Rewrite `comprehensive_test_suite.py` to dispatch real
      sub-suites and report actual numbers.
- [ ] Fix scoring inversion in `fuzz_test.py` (4 fuzzers).
- [ ] Add deterministic seeding option for reproducibility.

### Batch 2 — Import-time bugs
- [ ] Populate the empty fallback `UnitTypeId` in
      `scouting/advanced_scout_system_v2.py` with all attributes the
      module uses as defaults.
- [ ] Guard `check_learning_rate.py` and `check_proxy.py` with
      `if __name__ == "__main__":` so importing them is safe.

### Batch 3 — SC2Coach + tests
- [ ] Add 8+ new patterns to `sc2_coach.py` (queens, anti-air, creep,
      worker count, hatchery production, larva).
- [ ] Add `tests/test_sc2_coach.py` covering every pattern + statistics
      analysis branches.
- [ ] Wire the new test into `comprehensive_test_suite.py`.

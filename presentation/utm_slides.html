Boids 3D 군집 제어

SC2의 6-Force를 3D로 확장하고 고도 관련 2개 힘을 추가하여 총 8개 힘 벡터를 사용합니다.

Ftotal = Σi=18 wi · Fi(drone, neighbors, environment)

🌐 기존 SC2 힘 (3D 확장)

1. Separation w=2.0 / r=5m
역제곱 반발력, 드론 간 안전 거리 유지
2. Alignment w=1.0 / r=30m
이웃 속도 벡터 정렬, 편대 방향 통일
3. Cohesion w=1.0 / r=30m
그룹 중심 응집, 편대 유지
4. Target Seeking w=1.5
목표 웨이포인트로 이동, 거리 비례 감쇠
5. Obstacle Avoid w=2.5 / r=20m
장애물(건물, 산) 회피, 반발력 기반

⬆️ 신규 UTM 힘

6. Altitude Hold w=1.0
지정 고도 유지 (PID P항), 수직 복원력
7. Terrain Clearance w=3.0
최소 안전 고도(5m) 미만 시 강제 상승
8. Corridor Follow 동적
회랑 중심선 추적, 이탈 시 복원력

🎮 드론 운동학 제한

▸ max_speed: 15 m/s
▸ max_acceleration: 5 m/s²
▸ max_force: 5 m/s² (클램핑)
▸ altitude: 5~120m (법적 제한)

TTC 충돌 예측 시스템

기존 거리 기반 회피를 시간 기반 궤적 예측으로 업그레이드하여 선제적 충돌 회피를 구현합니다.

기존 방식 (SC2 Boids)

거리 기반 반발력
if distance < threshold → 반발력 적용

❌ 고속 접근 시 반응 지연
❌ 병렬 이동 드론 불필요한 회피
❌ 교차 궤적 사전 감지 불가

새 방식 (TTC 예측)

시간 기반 궤적 교차 예측
미래 위치 계산 → 최소 이격 거리 평가

✔ 5초 전 선제적 회피 기동
✔ 상대 속도 기반 정밀 판단
✔ 3단계 심각도 분류

📐 TTC 수학 모델

rel_pos = B.pos - A.pos
rel_vel = B.vel - A.vel

TTC = -dot(rel_pos, rel_vel)
      / dot(rel_vel, rel_vel)

min_dist = |A.pos + A.vel·TTC
          - (B.pos + B.vel·TTC)|

⚠️ 3단계 경보 체계

레벨TTC대응
WARNING≤ 5.0초경로 미세 조정
CRITICAL≤ 3.0초회피 기동 시작
IMMINENT≤ 1.0초긴급 회피 (수직)

가상 비행 회랑 시스템

SC2 점막 고속도로(CreepHighway)의 A* 경로를 3D 비행 회랑으로 진화시켰습니다.

✈️ 고도층 분리 (3 Layer)

LOW 5~30m — 촬영, 농업, 측량
MEDIUM 30~60m — 배송, 순찰, 감시
HIGH 60~120m — 장거리 이동, 중계

↔️ 방향별 고도 분리

▸ Eastbound (0~180°) → 기본 고도
▸ Westbound (180~360°) → +15m 오프셋
▸ 항공 교통 규칙 (ICAO) 모사

🛤️ 회랑 구조

FlightCorridor 구성요소:
▸ waypoints[] — 3D 웨이포인트 시퀀스
▸ width — 회랑 폭 (기본 20m)
▸ altitude_layer — 할당 고도층
▸ bidirectional — 양방향 여부

📈 고도 프로파일

0~15% 상승 구간 (이륙 → 순항 고도)
15~85% 순항 구간 (일정 고도 유지)
85~100% 하강 구간 (순항 → 착륙)

🚫 회랑 충돌 검출

두 회랑의 웨이포인트 교차점 자동 검출
→ 교차 지점에 시간 분리 규칙 적용
→ 동적 우회 경로 생성

Authority Mode — 긴급 대응 FSM

SC2 봇의 Authority Mode를 드론 공역 통제의 우선순위 기반 상태 머신으로 전환합니다.

🚨 EMERGENCY (P0)

트리거: TTC ≤ 1초, 배터리 부족, GPS 상실
대응: 즉시 정지, 수직 회피, 자동 귀환
SC2 대응: 적 러시 감지 → 전 유닛 방어

⚠️ DECONFLICT (P1)

트리거: TTC ≤ 5초, 회랑 교차
대응: 회피 기동, 속도/고도 조정
SC2 대응: 적 근접 → 전투 태세

📋 MISSION (P2)

트리거: 임무 수행 중
대응: 촬영/배송/순찰 임무 수행
SC2 대응: 전략 모드 → 테크 업그레이드

🛩️ CRUISE (P3)

트리거: 이동 중, 위협 없음
대응: Boids 편대 비행, 에너지 최적화
SC2 대응: 경제 모드 → 자원 수집

✅ IDLE (P4)

트리거: 대기 중, 착륙 상태
대응: 호버링 / 배터리 충전 대기
SC2 대응: BALANCED → 균형 유지

🔄 상태 전이 규칙

▸ 높은 우선순위가 항상 선점
▸ EMERGENCY는 모든 상태에서 즉시 전환
▸ 위협 해제 시 이전 상태로 복귀
▸ 배터리 20% 미만 → 강제 EMERGENCY

데이터 모델 & 드론 상태

📦 DroneState (= SC2 Unit)

@dataclass
class DroneState:
  id: int
  position: Point3D  # x,y,z (m)
  velocity: np.ndarray # [vx,vy,vz]
  heading: float    # rad
  drone_type: str   # quadrotor
  max_speed: float = 15.0
  max_accel: float = 5.0
  min_alt: float   = 5.0
  max_alt: float   = 120.0

📍 Point3D (= SC2 Point2)

class Point3D:
  x, y, z: float
  # distance_to() → 3D 유클리드
  # horizontal_distance_to() → 2D
  # to_array() → numpy 벡터
  # towards() → 방향 이동
  # offset() → 좌표 변위

🚨 CollisionAlert

class CollisionAlert:
  drone_a_id: int
  drone_b_id: int
  ttc: float        # 충돌까지 시간
  min_distance: float # 최소 접근 거리
  conflict_point: Point3D
  severity: str
   # "warning" | "critical" | "imminent"

🛤️ FlightCorridor

class FlightCorridor:
  corridor_id: str
  waypoints: List[Point3D]
  width: float = 20.0
  altitude_layer: str
   # "low" | "medium" | "high"
  bidirectional: bool

# 📱 GitHub 리포지토리 실시간 모니터링 가이드

## 🎯 개요

이 모바일 앱은 개인용으로 GitHub 리포지토리를 실시간으로 모니터링할 수 있는 PWA(Progressive Web App)입니다. 핸드폰으로 언제 어디서나 프로젝트의 상태를 확인할 수 있습니다.

## ✨ 주요 기능

### 1️⃣ 리포지토리 개요
- ⭐ Stars, 👁️ Watchers, 🔱 Forks 실시간 확인
- 최근 커밋 3개 요약
- 열린 Pull Request 목록
- 최신 릴리즈 정보

### 2️⃣ 커밋 모니터링
- 최근 커밋 목록 (최대 20개)
- 커밋 작성자, 시간, 메시지
- 커밋 SHA 해시
- 새로운 커밋 알림

### 3️⃣ Pull Request 추적
- 열린 PR 목록
- PR 제목, 번호, 작성자
- 코드 변경 통계 (+/-줄, 파일 수)
- 생성 시간

### 4️⃣ GitHub Actions/Workflows ⭐ NEW!
- 워크플로우 실행 상태 실시간 모니터링
- 성공 ✅ / 실패 ❌ / 진행중 ⏳ 표시
- 실행 번호, 브랜치, 이벤트 타입
- 워크플로우 링크로 바로 이동

### 5️⃣ 브랜치 관리 ⭐ NEW!
- 모든 브랜치 목록
- Protected 브랜치 표시
- 최신 커밋 SHA
- 브랜치 상태 확인

### 6️⃣ 릴리즈 추적
- 최신 릴리즈 정보
- Pre-release 표시
- 릴리즈 노트
- 다운로드 링크

### 7️⃣ 이슈 모니터링
- 열린 이슈 목록
- 이슈 번호, 제목, 작성자
- 라벨 표시 (색상 포함)
- 생성 시간

## 🚀 빠른 시작

### 1단계: 앱 접속
```
https://5174-inm4xsfrf1dh760iaux6u-02b9cc79.sandbox.novita.ai
```

### 2단계: 저장소 설정
1. GitHub 페이지의 **"설정"** 버튼 클릭
2. **소유자**(Owner)와 **저장소**(Repository) 이름 입력
   - 예: `sun475300-sudo` / `Swarm-control-in-sc2bot`
3. **저장** 버튼 클릭

### 3단계: GitHub 토큰 설정 (권장) ⭐ NEW!

#### 왜 필요한가?
- **인증 없이**: API Rate Limit 60회/시간 ⚠️
- **토큰 사용 시**: API Rate Limit 5000회/시간 ✅

#### 토큰 생성 방법
1. Settings 페이지로 이동
2. "GitHub 개인 액세스 토큰" 섹션 찾기
3. **"토큰 생성하기 →"** 링크 클릭
4. GitHub 설정 페이지에서:
   - **Note**: `모바일 모니터링 앱` (원하는 이름)
   - **Expiration**: `No expiration` 또는 원하는 기간
   - **Scopes**: 
     - ✅ `repo` (프라이빗 리포지토리용)
     - ✅ `public_repo` (퍼블릭 리포지토리만 사용 시)
     - ✅ `read:org` (조직 리포지토리용, 선택사항)
5. **Generate token** 클릭
6. 생성된 토큰 복사 (한 번만 표시됨!)
7. 모바일 앱의 Settings로 돌아가서 토큰 입력
8. **토큰 저장** 버튼 클릭

#### 토큰 유효성 확인
- ✅ 녹색 체크: 토큰이 유효함
- ❌ 빨간 X: 토큰이 유효하지 않음
- API Rate Limit 사용량 실시간 표시

### 4단계: 홈 화면에 추가 (PWA)
1. 브라우저 메뉴에서 **"홈 화면에 추가"** 선택
2. 앱 아이콘이 핸드폰 홈 화면에 추가됨
3. 앱처럼 사용 가능!

## 🔒 보안 및 프라이버시

### 데이터 저장
- 모든 설정은 **브라우저 로컬 스토리지**에 저장
- GitHub 토큰은 **암호화되지 않은 상태**로 로컬 저장
- 외부 서버로 전송되지 않음

### 권장 사항
- 퍼블릭 컴퓨터에서 사용 시 주의
- 토큰은 **최소 권한**만 부여 (read-only)
- 정기적으로 토큰 갱신

## 🐛 문제 해결

### "대시보드에 연결할 수 없습니다"
- 대시보드 URL 확인
- 네트워크 연결 확인
- 서버 상태 확인

### "API Rate Limit 초과"
- GitHub Personal Access Token 설정
- Settings → GitHub 토큰 섹션
- 토큰 설정 후 5000/hour 사용 가능

### "토큰이 유효하지 않습니다"
- 토큰 재생성 필요
- GitHub에서 토큰 삭제 후 재발급
- Scope 확인 (repo 권한 필요)

### 데이터가 표시되지 않음
- 저장소 설정 확인
- 브라우저 캐시 삭제
- 페이지 새로고침 (Pull to refresh)

### 알림이 작동하지 않음
- 브라우저 알림 권한 확인
- Settings에서 알림 활성화 확인
